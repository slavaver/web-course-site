**Цель:** освоить работу с объектно-ориентированным программированием в TS. Изучить работу с объединениями, пересечениями объектных типов, классами и интерфейсами. Освоить утверждение типа при работе с сетевыми запросами.

## Подготовка

Изучите работу с объектными типами, их объединением и пересечением.

Изучите работу в TS с классами, абстрактными классами, их расширением и реализацией интерфейсов. Посмотрите на модификаторы доступа к полям классов.

Посмотрите как делается проверка входящих данных из сетевых запросов, так как Response.json возвращает тип any.

Повторить информацию о Fetch API, http протоколе (сообщения, запросы, ответы, статусы, методы) и REST подходе.

Прочитайте про [хуки жизненного цикла](https://ru.vuejs.org/guide/essentials/lifecycle.html), чтобы определить в какой момент отправлять сетевой запрос на получение данных.

Изучите передачу информации по иерархии компонентов: через props и emit, слоты (slots) для передачи готовой разметки.

Установите [json-server](https://www.npmjs.com/package/json-server) он будет вашей заменой сервера с API. Просмотрите документацию проекта, чтобы примерно знать, что она вам дает.

## Выполнение

В этой работе предлагается реализовать часть объектов и их взаимодействие из LMS (системы управления обучением).

Объекты:

- Пользователи. Тип пользователя с идентификатором, который можно только читать, именем и электронной почтой
- Студент. Имеет дополнительное поле "год поступления".
- Преподаватель. Имеет дополнительное поле "ученая степень": отсутствует, к.н., д.н..
- Курсы. Тип курса с идентификатором только на чтение, названием, опциональным **лимитом студентов на одного преподавателя**, статусом (черновик, архивный, активный), и списками записанных студентов и преподавателей. Методы для публикации и архивации курса. Метод для добавления пользователя к курсу.

Подумайте над тем как лучше организовать код в файлах. Код предметной области лучше держать отдельно от компонентов Vue, чтобы интерфейс и реактивность Vue взаимодействовали с экземплярами объектов, а ограничения предметной области проверялись в самих объектах.

Подготовьте пользовательский интерфейс в Lab2View.vue. Повторяющиеся элементы интерфейса выделяйте в отдельные компоненты в соответствующей папке.

Информация о пользователях и курсах должна храниться в файле db.json, с которым работает json-server. В этой работе вам нужно будет реализовать взаимодействие с REST API, используя Fetch API. Вам потребуется добавлять и удалять пользователей, подумайте над тем относятся ли методы удаления и добавления пользователей/курсов к самим объектам или стоит выделить их в другой класс.

На странице пользователь может:

- добавлять и удалять студентов и преподавателей.
- просматривать информацию о всех пользователях в алфавитном порядке.
- создавать и удалять курсы.
- просматривать информацию о курсах.
- записывать и удалять пользователей с курсов.
- просмотреть среднее количество студентов на всех курсах.
- просмотреть количество кандидатов и докторов наук ведущих курсы.

## Требования

- Проект собирается, отформатирован и нет замечаний от ESLint и tsc, нет ошибок в консоли браузера.

Интерфейс должен быть аккуратным:

- связанные элементы находятся друг к другу ближе, чем к другим;
- элементы выровнены;
- схожие элементы имеют одинаковый размер;
- интерфейс понятен;
- интерфейс отзывчив и располагается в центре экрана, без выключки текста по центру; не расползается на весь экран.

Для стилизации можете использовать сторонние библиотеки/фреймворки, можете писать стили самостоятельно в компонентах (CSS, SCSS). Предпочтительнее вариант самостоятельной работы и посмотрите на директиву css [@container](https://developer.mozilla.org/en-US/docs/Web/CSS/@container) для отзывчивости.
